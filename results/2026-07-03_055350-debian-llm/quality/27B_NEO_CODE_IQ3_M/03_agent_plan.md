# 27B_NEO_CODE_IQ3_M — 03_agent_plan

## Prompt

You are an agent in a sandbox. Give a step-by-step plan to find every TODO comment in a Git repo, then a single bash command that does it.

## Response

### Step-by-Step Plan
1. **Navigate to the repository root**: Open your terminal and `cd` into the root directory of the Git repository you want to search.
2. **Verify the working tree**: Ensure you're on the correct branch and that your index is up-to-date (`git status` or `git pull` if needed).
3. **Choose the right search scope**: Decide whether you want to search only tracked files (recommended for Git repos) or include untracked/local files.
4. **Apply case-insensitive matching**: Use a flag to catch `TODO`, `todo`, `Todo`, etc.
5. **Enforce word boundaries**: Prevent false positives like `TODOLIST` or `TODAY` by matching `TODO` as a standalone word.
6. **Execute and parse output**: Run the search. The output will print `filepath:linenumber:matching_line` for every hit, making it easy to navigate or pipe into other tools.

### Single Bash Command
```bash
git grep -i -w "TODO"
```

**Flag breakdown:**
- `git grep`: Optimized, recursive search across tracked Git files (ignores `.gitignore` patterns and binary files automatically).
- `-i`: Case-insensitive matching.
- `-w`: Match only whole words (prevents matching substrings like `TODOLIST`).

**Optional tweaks:**
- Include submodules: `git grep -i -w "TODO" --recurse-submodules`
- Search untracked files too: `grep -rnwi "TODO" --include="*.*" .`
- Grep-style regex for common comment prefixes: `git grep -i -E
